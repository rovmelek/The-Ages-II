## Epic Summary

Epic 16 was the prerequisite for Godot client development — formalizing the WebSocket protocol, hardening server architecture, and adding resilience features (session tokens, grace period, message acknowledgment). The epic was organized into 4 sprints with a clear dependency graph:

- **Sprint 1 (Protocol):** 16.1, 16.2, 16.3, 16.12 — Pydantic schemas for all 21 inbound + 38 outbound message types, auto-generated protocol spec document, chat markdown support
- **Sprint 2 (Refactoring):** 16.4a, 16.4, 16.5, 16.6, 16.10a — grant_xp decomposition, combat service layer extraction, login decomposition, TradeManager constructor injection, combat turn timeout enforcement
- **Sprint 3a (Client Polish):** 16.7, 16.8 — request-response correlation (request_id), heartbeat/connection health
- **Sprint 3b (Resilience):** 16.9, 16.10, 16.11 — session tokens for reconnection, disconnected player grace period, message acknowledgment IDs

## What Went Well

1. **Story 16.5's refactor investment paid off immediately.** The 5 helpers extracted from `handle_login` were directly reused by `handle_reconnect` in Story 16.9 — same-epic ROI for a refactoring story.

2. **Stub fields pattern worked cleanly across story boundaries.** Story 16.9 added `disconnected_at`, `connected`, and `_cleanup_handles` as stubs. Story 16.10 populated them. Zero rework, clean two-story dependency handoff.

3. **Adversarial review loop caught real bugs before implementation.** Every story went through 2-5 adversarial passes. Caught: wrong function names (16.11), dangerous spec wording (16.10 "replaces" vs "precedes"), dict aliasing bugs (16.11), missing early-return guard removal (16.11).

4. **Additive field strategy prevented test cascades.** `seq`, `request_id`, `format`, `connected`, `session_token`, `turn_timeout_at` — all added as optional with `None` default. Existing tests never broke from new fields.

5. **The `autouse` fixture strategy (16.10) was critical.** Adding `DISCONNECT_GRACE_SECONDS=0` before any code changes preserved immediate-cleanup behavior for all existing tests.

6. **Tech spec hardening paid dividends.** 9 Red Team rounds, pre-mortem, war room, and 7 ADRs before implementation meant zero architectural surprises during execution.

---

## Challenges and Learnings

1. **Signature threading ripple (16.7).** The `data` dict parameter had to be threaded through `_handle_exit_transition` and party sub-handlers — not anticipated in the spec. 15 files touched (broadest footprint in the epic).

2. **Test mock cascades across stories.** Stories 16.10 and 16.11 required updating test fixtures from earlier stories (heartbeat tests needed `_shutting_down`/`_cleanup_handles` attrs; xp/level-up/trade tests needed `send_to_player_seq = AsyncMock()`).

3. **`Game.__new__` pattern in tests is fragile.** Tests that skip `__init__` via `Game.__new__` break when new fields are added to `Game.__init__`. Each new field requires retroactive test fixes.

4. **Line numbers in story specs go stale.** Carried forward from Epic 15 retro — partially addressed (adversarial passes catch stale refs) but not eliminated.

## Key Insights

1. **Adversarial review at the spec level is the highest-value quality gate.** Code review typically found 0-3 issues per story because the spec was already verified. 80% of bugs prevented at spec level, not code level.

2. **Stub fields + sequential story chains work.** 16.9 → 16.10 → 16.11 proved that adding stubs in story N and populating in N+1 is a clean dependency pattern with zero coordination overhead.

3. **`Game.__new__` in tests is recurring tech debt.** Every new `Game` attribute breaks tests that skip `__init__`. A shared mock factory would eliminate this.

4. **Refactor-first stories have measurable ROI within the same epic.** 16.5 enabled 16.9. 16.4a enabled clean `send_to_player_seq` adoption in 16.11.

5. **The autouse fixture pattern is essential for behavior-changing features.** Adding the test safety net before any code changes (not after) prevents the "all tests break" scenario.

## Architecture Decisions Record (Epic 16)

- **ADR-16-1:** WebSocket retained as transport (Godot+Unity+Unreal+web+FastAPI compatibility)
- **ADR-16-2:** In-memory TokenStore (300s TTL too short for DB; server restart kills all state anyway)
- **ADR-16-3:** `msg_seq` on ConnectionManager, not PlayerSession (networking concern, not session concern)
- **ADR-16-4:** Server client-agnostic for chat — NO HTML stripping (each client handles own rendering security)
- **ADR-16-5:** `PlayerEntity.connected` field (not `get_state()` signature change — avoids touching call sites)
- **ADR-16-6:** Deferred cleanup via `call_later` + public `PlayerManager.deferred_cleanup()` (matches TradeManager pattern)
- **ADR-16-7:** `_default_stats()` as function, not constant (supports test monkeypatching)
