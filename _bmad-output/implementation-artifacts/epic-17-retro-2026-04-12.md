## Epic Summary

Epic 17 addressed findings from a comprehensive adversarial codebase review. The goal was to centralize scattered constants, extract business logic from fat handlers into service modules, and add structured error codes. Organized into 3 sprints:

- **Sprint 1 (Constants & Config):** 17.1-17.5 — StrEnum type constants, game-balance fallback centralization, direction string unification, TradeState enum, asyncpg portability
- **Sprint 2 (Service Extraction):** 17.7, 17.6, 17.9, 17.10, 17.8 — auth service + session deduplication, XP notification layer separation, combat initiation/cleanup extraction, trade execution extraction, Game class decomposition
- **Sprint 3 (Protocol & Polish):** 17.11-17.16 — structured error codes, Pydantic error sanitization, protocol versioning, auto-generated help categories, data-driven handler registration, markdown line break rendering

## What Went Well

1. **StrEnum (ADR-17-1) enabled zero-test-change constant centralization.** `EffectType`, `TradeState`, `ErrorCode` all compare equal to plain strings — 23 consumer sites updated with zero test breakage.

2. **Thin delegation wrappers (ADR-17-3) preserved mock surfaces.** `Game.kill_npc`, `Game.respawn_player`, heartbeat methods — all moved out without breaking any test that mocks them.

3. **Sprint 2 was cleanly parallelizable.** Stories 17.6, 17.9, 17.10 had zero code dependencies. Sequential ordering (17.7 first, 17.8 last) eliminated merge conflicts.

4. **Adversarial review loop caught real issues.** Story 17.7 went through 4 passes finding 24 issues (missing call sites, incorrect patch tables, omitted steps). All caught before implementation.

5. **`core/xp.py` zero net imports** validates the layering goal — core is pure business logic.

6. **ADR-17-5 scope discipline** kept Stories 17.11/17.12 focused on exactly 5 protocol/auth errors, avoiding scope creep into ~70 handler-level error sites.

---

## Challenges and Learnings

1. **Dual-patch pattern (Story 17.7):** When `player_repo` is imported by both `auth.py` and `service.py`, tests must patch both with the same mock. Non-obvious pattern that should be documented.

2. **Lambda late-binding (Story 17.15):** `h=handler` default arg needed to avoid closure bug. Well-known Python gotcha, easy to miss.

3. **Line numbers in specs remain friction.** Carried forward from Epic 15 and 16. Function/class names are more stable references.

4. **Story 17.7 was oversized** (~10 files, ~30 patch targets). Consider splitting stories of this complexity in future.

---

## Key Insights

1. **Adversarial spec review is the highest-value quality gate for refactoring epics.** Patch target verification at spec level prevents implementation failures.

2. **StrEnum + thin delegation = ideal incremental refactoring pattern.** Both preserve backward compatibility while improving architecture.

3. **Service extraction is predictable work.** Risk is entirely in patch target migration, not logic errors.

4. **`make_bare_game()` validated the Epic 16 retro investment.** Essential for Stories 17.7 and 17.8.

5. **CSS `white-space: pre-wrap` (17.16)** — finding the simpler solution that fixes both rendering paths simultaneously.
