## What Went Well

### 1. Multi-Pass Adversarial Review Loops
Enforcing minimum 2 passes on both story spec reviews and code reviews caught real issues — wrong line numbers, stale function references, missing XP double-count analysis, undeclared `_lastXp` state variable. These would have caused dev agent errors or subtle bugs if not caught pre-implementation.

### 2. Investigation Over Assumption
The 2 "known hanging" tests (`test_disconnect_notifies_others`, `test_register_returns_player_id`) had been excluded since Epic 6. Kevin pushed for root cause investigation rather than accepting the status quo. This revealed the `async_session` patching gap — the same structural flaw that caused the `TestChestInteraction` failures from Epic 10. One investigation resolved three separate long-standing issues.

### 3. Session Factory Dependency Injection (Story 11.7)
Moving DB session factory ownership to `Game` eliminated an entire category of test failures. The old pattern (patching `async_session` in each consumer module) required 67 patches across 14 test files and silently broke whenever a new module was added. The new pattern (`game.session_factory = test_factory`) is one assignment, and it's structurally impossible to miss a module.

### 4. Clean XP Architecture (Stories 11.3-11.4)
Creating `grant_xp()` as a single shared function for all XP sources (combat, exploration, interaction) with automatic CHA bonus, DB persistence, WebSocket notification, and level-up detection gave universal coverage. Adding exploration XP and interaction XP required zero changes to the core function.

### 5. Incremental Story Progression
The epic built cleanly from foundation (11.1 stats) through integration (11.2 combat) to systems (11.3-11.5 XP/leveling) to UI (11.6) to infrastructure (11.7). Each story could reference concrete patterns from the previous one, and the "Previous Story Intelligence" sections in story files enabled effective knowledge transfer.

### 6. Test Suite Fully Clean
For the first time in the project's history, all 601 tests pass with zero exclusions. The 4 long-standing test failures (2 hanging, 2 chest) are all resolved.

## Challenges & Lessons Learned

### 1. Never Accept "Known Issues" Without Root Cause
The hanging tests persisted through 4 epics because they were documented and excluded rather than investigated. The root cause (unpatched `async_session` → real DB writes → SQLite lock contention from zombie processes) was straightforward once investigated.

**Lesson:** The longer an assumption persists, the more likely it masks a structural problem. Investigate before excluding.

### 2. Mock Factory Semantics: MagicMock vs AsyncMock
`AsyncMock()` as a session factory returns a coroutine, not an async context manager. The correct pattern is `MagicMock(return_value=mock_ctx)` — sync callable returning async context manager. This caused multiple test failures during Story 11.7 until the pattern was understood.

**Lesson:** `session_factory()` is a sync call. Document the correct mock pattern.

### 3. Integration Test Fragility with Fixed receive_json() Counts
New `xp_gained` messages from Stories 11.3-11.4 broke integration tests that expected a fixed number of messages. The `_drain_until(msg_type)` helper pattern proved resilient.

**Lesson:** Prefer `_drain_until(msg_type)` over fixed-count `receive_json()` in integration tests.

### 4. Large Refactor Blind Spots (Story 11.7)
The initial pass of Story 11.7 missed 4 test files (`test_blocking_objects.py`, `test_lever.py`, `test_events.py`, `test_spawn.py`) that patched the removed `async_session` attribute. These were caught by running the test suite, not by the story's file list.

**Lesson:** When removing a module attribute, `grep -r` all test files — not just the ones in the story spec.

### 5. XP Double-Count Risk (Story 11.6)
The web client's `combat_end` handler manually incremented XP, and the new `xp_gained` handler also incremented it. Both fire for combat victories.

**Lesson:** When adding a new handler that updates shared state, audit all existing handlers touching the same state.
