## Epic Summary

Epic 15 addressed 7 findings from an adversarial codebase review (2026-04-11), tightening the server's internal structure through pure refactoring. The epic was organized into two phases: Phase 1 (sequential, 15.1-15.3) established core infrastructure (PlayerManager, cleanup relocation, auth middleware), and Phase 2 (independent, 15.4-15.7) resolved remaining findings in parallel-safe order.

## What Went Well

1. **Phased execution strategy.** Phase 1 (sequential: 15.1 → 15.2 → 15.3) established foundations that Phase 2 (independent: 15.4-15.7) built on without conflicts. No rework needed between stories.

2. **Adversarial review loop.** Every story file went through 2-3 adversarial passes before implementation, catching wrong line numbers, inconsistent parameter names, redundant API designs, and stale references. This prevented bugs before code was written.

3. **Test suite stability.** Zero assertion value changes across 7 stories. The only test addition was the EventBus error isolation test (Story 15.6). No flaky tests introduced.

4. **Blast radius management.** Story 15.1 (13 server files + 26 test files), Story 15.5b (16 import sites + file deletions), and Story 15.3 (10 handler files) all landed cleanly despite large blast radii.

5. **Systematic debt elimination.** Every finding from the adversarial codebase review was resolved: raw dict → PlayerManager, reverse dependency → cleanup relocation, boilerplate → decorator, module-level state → manager attributes, code duplication → shared method, inline queries → repo pattern.

---

## Challenges and Learnings

1. **Hidden import coupling (Story 15.5b).** Removing the `Party` class from `manager.py` accidentally removed `import time` that `PartyManager.set_cooldown()` depended on. 5 tests failed. Lesson: when relocating classes between files, check the entire source file for side-effect imports that other code in the file relies on.

2. **Pre-existing flaky test.** `test_integration.py` has an intermittent failure (different test fails each run, passes in isolation). Noted in Story 15.6 dev log. This predates Epic 15 and should be investigated.

3. **Story file line numbers go stale.** Each story's line references were accurate at creation time but shifted as previous stories modified the same files. The adversarial review loop caught these before implementation. Future improvement: prefer function/class names over line numbers in story specifications.

## Key Insights

1. **Pure refactoring epics benefit enormously from adversarial story review.** The story-level adversarial pass (before code review) is where most real issues were caught. The code review pass typically found 0 issues because the story file was already verified.

2. **API naming requires scrutiny during story creation.** Story 15.7's initial design had a redundant get+upsert pattern caught by adversarial review. Naming decisions (upsert vs create, kwargs vs explicit params) deserve deliberate attention.

3. **Constructor injection is the consistent pattern for manager dependencies.** `PartyManager(connection_manager=...)` mirrors `CombatManager(effect_registry=...)`. This should be the default for any future managers.

4. **ADR-15-8 (defer upsert unification) was the right call.** The three upsert patterns across repos differ enough in field mapping that a generic utility would have been premature abstraction.

## Architecture Decisions Record (Epic 15)

- **ADR-15-1:** PlayerManager is a plain class (mirrors PartyManager/TradeManager style)
- **ADR-15-2:** `cleanup_session(entity_id, game)` takes full Game object (cleanup touches 5 managers)
- **ADR-15-3:** `@requires_auth` in dedicated `auth_middleware.py`, outer function retains original signature
- **ADR-15-4:** Party invite dicts become PartyManager attributes, connection_manager via constructor injection
- **ADR-15-5:** NpcEntity at `server/room/npc.py` (room-spatial but not InteractiveObject)
- **ADR-15-6:** EventBus error isolation via try/except per callback (log and continue, no retry)
- **ADR-15-7:** `mob_states` removed (never used), Alembic migration drops column
- **ADR-15-8:** Upsert unification deferred (premature abstraction)
